## 1. 启动与置顶窗口

### 验收步骤

1. 双击 `异环自动钓鱼.exe`。
2. 检查窗口尺寸、拖动、最小化、置顶和关闭。
3. 检查目标次数仅允许 1～999。
4. 尝试在运行中修改次数。

### 预期与证据

- 窗口正常打开且无控制台黑窗。
- 窗口模式和无边框游戏下始终置顶；独占全屏按设计说明处理。
- 运行中目标次数被锁定。
- 证据：启动冒烟测试记录和人工截图。

### 结果记录

- 状态：发布物启动、响应、关闭和 Windows GUI 子系统离线检查通过；400×240 初始/最小宽度及右侧阶段/帧率列间距自动测试通过；窗口拖动、最小化、置顶、次数边界及真实屏幕上的文字无裁切为人工确认（未执行）。
- 反馈/问题：`try/smoke_exe.ps1` 输出 `SMOKE_OK` 且本次拥有的进程树残留数为 0；并发隔离输出 `CONCURRENT_OWNERSHIP_OK`，其他同路径实例未被检查、关闭或终止。PE 子系统值为 2，未显示控制台子系统。独占全屏可见性仍按设计限制人工确认。

## 2. 游戏窗口绑定

### 验收步骤

1. 设置次数后点击“绑定并开始”，在三秒内切换到《异环》；确认倒计时结束后直接启动，不再点击第二次“开始”。
2. 分别使用窗口、无边框和全屏。
3. 移动或调整游戏窗口尺寸后重新开始。
4. 尝试绑定无效窗口、最小化窗口和过小窗口。

### 预期与证据

- 显示正确窗口标题、客户区尺寸和绑定状态，并在同一次倒计时后直接启动。
- 分辨率或位置变化后重新计算客户区，不使用固定 2560×1440 坐标。
- 运行时至少每 0.5 秒复核客户区位置和尺寸。
- 无效状态安全暂停并显示具体原因。
- 证据：窗口坐标自动测试、状态日志和实机人工确认。

### 结果记录

- 状态：窗口坐标、DPI、顶层 HWND 解析、显示器选择、一次倒计时绑定并启动及 F8 取消路径自动测试通过；窗口、无边框和全屏绑定为人工确认（未执行）。
- 反馈/问题：应用已统一使用 `GetAncestor(..., GA_ROOT)` 得到的顶层 HWND 识别自身及排除捕获；排除失败会显示非致命警告。结构扫描确认生产代码没有固定 2560×1440 坐标；该分辨率仅作为多分辨率测试参数。

## 3. 单轮完整流程

### 验收步骤

1. 在钓鱼就绪画面设置目标 1 次并开始。
2. 观察第一次 F、上钩识别、第二次 F、进度条控制、顶部进度条消失和随机延迟后的结算点击。
3. 检查完成计数。

### 预期与证据

- 顺序与设计状态机一致，不跳过状态。
- 顶部进度条至少稳定 15 个控制帧并连续三帧干净消失后，随机等待 3.10～3.60 秒点击一次安全空白点；鱼获页内容不参与判断。
- 结算点击的输入调用正常返回后，完成数量立即从 0 变为 1，不再等待或识别后续画面；输入调用抛错时保持 0 并安全暂停。
- 证据：状态迁移测试、简短运行日志和实机人工确认。

### 结果记录

- 状态：稳定消失门槛、随机结算点击和多轮状态迁移自动测试通过；用户已确认真实多轮执行完整闭环，当前仅需复验上钩识别稳定性。
- 反馈/问题：初次使用只进行“绑定并开始”的一次 3、2、1 倒计时；已绑定后的“开始”和暂停后的“继续”默认自动激活游戏，关闭“自动切回游戏”后恢复手动倒计时。激活失败或激活后仍非前台时不得启动/恢复或发送输入；自动轮间抛竿不得激活窗口。2.10～2.60 秒版本实机未能关闭最后页面，当前仍使用 3.10～3.60 秒；点击输入正常返回即计数，不再验证结算画面。

## 4. 每秒 30 帧闭环控制

### 验收步骤

1. 运行合成帧回放，绿色区域和黄色标记以不同速度和方向移动。
2. 验证黄色标记偏左、偏右和位于安全带时的输入。
3. 分别插入仍有扫描线/候选的结构性歧义 59～60 帧，以及零扫描线/零候选的完全空白 59～60 帧；另验证三帧干净 `yellow_missing` 仍进入结算。
4. 在真实游戏快速变化场景中观察实际帧率和控制结果。

### 预期与证据

- 偏左短按 D，偏右短按 A，进入内缩安全带释放两键。
- 换向时先释放旧键，任意时刻不同时按 A/D。
- 一帧缺失立即释放 A/D；结构性歧义和非干净完全空白均在第 60 帧暂停，恢复可靠观测或切换类别后重新计数；稳定控制后的三帧干净消失仍优先进入结算。
- 控制线程消费最新帧，不积压回放旧帧。
- 证据：`python -m pytest try/tests -q`、回放统计和实机人工确认。

### 结果记录

- 状态：30 帧最新帧、黄标切断绿条重建、黄标与绿区完全分离、五行多数确认、方向、换向释放、中心死区、1.2% 最小窄绿区、稳定后干净消失和最长 59 帧漏识别后恢复的自动测试通过；真实失败原图回放输出绿区约 49.7%～59.0%、黄标约 66.7%。新发布物的高品质短绿区连续跟随为人工确认（未执行）。
- 反馈/问题：`20260713T110428410513Z_E_PROGRESS_LOST.png` 中真实绿区仍约占顶部 ROI 9.3%，但黄标已位于其右侧约 7.7% 槽宽；旧识别器只生成相邻/重叠配对，因而误报 `bar_too_narrow` 后变为 `no_consensus`。现在绿区和黄标在整个顶部槽带可玩区独立检测，历史仅分别用于同色候选择优，达到最小宽度的短绿区优先于附近噪声；非干净空白与结构歧义均最多等待 60 帧并全程释放 A/D，至少 15 个可靠控制帧后的连续三帧干净 `yellow_missing` 仍优先进入定时结算。实机需记录实际帧率、成功数及每次暂停原因。

## 5. 暂停、F8 与异常保护

### 验收步骤

1. 分别在等待上钩、闭环控制和结算阶段点击暂停或按 F8。
2. 将游戏切出前台、最小化或关闭。
3. 模拟截图停滞、识别超时和输入失败。
4. 检查恢复行为。

### 预期与证据

- 所有路径立即释放 F/A/D/鼠标左键且停止新输入。
- 错误原因显示在界面并记录一次诊断。
- 恢复前重新分类画面，无法分类时保持暂停。
- 证据：按键记录测试、状态机测试和实机人工确认。

### 结果记录

- 状态：F8、失去前台、截图停滞、超时、输入失败、恢复前手动前台复核、跨帧恢复令牌、迟到窗口错误、取消当前轮后重绑和退出释放的自动测试通过；真实游戏中的立即释放和恢复为人工确认（未执行）。
- 反馈/问题：暂停且无恢复令牌时 worker 在截帧/窗口/视觉边界前等待；阻塞中的旧错误不得清除新恢复令牌。默认“继续”会显式激活已绑定游戏并复核前台，关闭自动切回后才显示 3、2、1 手动倒计时。F8 会取消待绑定并开始、待开始和待继续倒计时，过期回调不得绑定、启动、发送输入或恢复 worker。窗口失效后可停止工作线程与截屏、释放输入、重新绑定并再次开始。
