## 1. 启动与置顶窗口

### 验收步骤

1. 双击 `异环自动钓鱼.exe`。
2. 检查窗口尺寸、拖动、最小化、置顶和关闭。
3. 检查目标次数仅允许 1～999。
4. 尝试在运行中修改次数。

### 预期与证据

- 窗口正常打开且无控制台黑窗。
- 窗口模式和无边框游戏下始终置顶；独占全屏按设计说明处理。
- 运行中目标次数被锁定。
- 证据：启动冒烟测试记录和人工截图。

### 结果记录

- 状态：发布物启动、响应、关闭和 Windows GUI 子系统离线检查通过；400×240 初始/最小宽度及右侧阶段/帧率列间距自动测试通过；窗口拖动、最小化、置顶、次数边界及真实屏幕上的文字无裁切为人工确认（未执行）。
- 反馈/问题：`try/smoke_exe.ps1` 输出 `SMOKE_OK` 且本次拥有的进程树残留数为 0；并发隔离输出 `CONCURRENT_OWNERSHIP_OK`，其他同路径实例未被检查、关闭或终止。PE 子系统值为 2，未显示控制台子系统。独占全屏可见性仍按设计限制人工确认。

## 2. 游戏窗口绑定

### 验收步骤

1. 点击“绑定游戏”，在三秒内切换到《异环》。
2. 分别使用窗口、无边框和全屏。
3. 移动或调整游戏窗口尺寸后重新开始。
4. 尝试绑定无效窗口、最小化窗口和过小窗口。

### 预期与证据

- 显示正确窗口标题、客户区尺寸和绑定状态。
- 分辨率或位置变化后重新计算客户区，不使用固定 2560×1440 坐标。
- 运行时至少每 0.5 秒复核客户区位置和尺寸。
- 无效状态安全暂停并显示具体原因。
- 证据：窗口坐标自动测试、状态日志和实机人工确认。

### 结果记录

- 状态：窗口坐标、DPI、顶层 HWND 解析、显示器选择和异常路径自动测试通过；窗口、无边框和全屏绑定为人工确认（未执行）。
- 反馈/问题：应用已统一使用 `GetAncestor(..., GA_ROOT)` 得到的顶层 HWND 识别自身及排除捕获；排除失败会显示非致命警告。结构扫描确认生产代码没有固定 2560×1440 坐标；该分辨率仅作为多分辨率测试参数。

### 结果记录

- 反馈/问题：点击“开始”后会显示“请在倒计时结束前切回已绑定的游戏窗口”和 3、2、1 倒计时，用户必须在倒计时内手动切回已绑定游戏。结束时游戏在前台才启动截屏和 worker；否则不按键、不启动并显示同一可重试错误。自动化不会在首帧或后续每轮调用 `SetForegroundWindow`。控制阶段只有原始结算候选连续两帧才进入等待结算，最终点击仍等待三帧确认；自动证据不能证明真实画面阈值有效，必须实机保存日志与结果。

## 4. 每秒 30 帧闭环控制

### 验收步骤

1. 运行合成帧回放，绿色区域和黄色标记以不同速度和方向移动。
2. 验证黄色标记偏左、偏右和位于安全带时的输入。
3. 插入单帧和连续六帧识别缺失。
4. 在真实游戏快速变化场景中观察实际帧率和控制结果。

### 预期与证据

- 偏左短按 D，偏右短按 A，进入内缩安全带释放两键。
- 换向时先释放旧键，任意时刻不同时按 A/D。
- 一帧缺失立即释放；连续约 0.2 秒缺失则暂停。
- 控制线程消费最新帧，不积压回放旧帧。
- 证据：`python -m pytest try/tests -q`、回放统计和实机人工确认。

### 结果记录

- 状态：30 帧最新帧、方向、换向释放、安全带、结算候选优先级和识别缺失自动测试通过；真实游戏快速变化连续 5 轮为人工确认（未执行）。
- 反馈/问题：5 帧普通缺失后的第 6/7 帧连续结算候选会进入等待结算，不会被普通丢失抢先暂停；第 6 帧单候选在第 7 帧中断则以 `E_PROGRESS_LOST` 暂停。实机需记录实际帧率、成功数及每次暂停原因，不得用固定延时绕过识别失败。

## 5. 暂停、F8 与异常保护

### 验收步骤

1. 分别在等待上钩、闭环控制和结算阶段点击暂停或按 F8。
2. 将游戏切出前台、最小化或关闭。
3. 模拟截图停滞、识别超时和输入失败。
4. 检查恢复行为。

### 预期与证据

- 所有路径立即释放 F/A/D/鼠标左键且停止新输入。
- 错误原因显示在界面并记录一次诊断。
- 恢复前重新分类画面，无法分类时保持暂停。
- 证据：按键记录测试、状态机测试和实机人工确认。

### 结果记录

- 状态：F8、失去前台、截图停滞、超时、输入失败、恢复前手动前台复核、跨帧恢复令牌、迟到窗口错误、取消当前轮后重绑和退出释放的自动测试通过；真实游戏中的立即释放和恢复为人工确认（未执行）。
- 反馈/问题：暂停且无恢复令牌时 worker 在截帧/窗口/视觉边界前等待；阻塞中的旧 `latest()` 迟到失败、陈旧帧 `release_inputs()` 迟到失败和旧 `E_WINDOW` 都不得递增新 epoch、清除新恢复令牌或退出 worker。点击“继续”会显示 3、2、1 倒计时，用户手动切回游戏后仅复核前台；失焦保持 `PAUSED/E_WINDOW` 且不生成令牌。F8 会取消待开始和待继续倒计时，过期回调不得发送输入或恢复 worker。窗口失效后可停止工作线程与截屏、释放输入、重新绑定并再次开始。实机仍需在控制阶段按 F8，并检查 A/D 实际释放。
