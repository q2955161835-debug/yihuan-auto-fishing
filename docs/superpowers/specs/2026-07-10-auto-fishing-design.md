# 《异环》自动钓鱼程序设计规格

## 1. 文档状态

- 日期：2026-07-10
- 状态：四部分设计已由用户确认，等待用户复核本书面规格。
- 目标平台：64 位 Windows 10/11。
- 交付形式：单个 `异环自动钓鱼.exe`，无需用户安装 Python。

## 2. 目标与范围

程序提供一个类似现有自动麻将工具的小型置顶窗口。用户绑定《异环》游戏窗口、设置钓鱼次数并点击开始后，程序按循环完成：

1. 按 F 开始钓鱼。
2. 识别右下角上钩提示变化，再按 F 收杠。
3. 以 30 帧/秒识别顶部绿色安全区和黄色标记，通过 A/D 保持黄色标记位于绿色区域内。
4. 识别结算画面并点击安全空白区域关闭。
5. 确认返回钓鱼就绪画面后计数并开始下一轮。

程序支持不同分辨率、Windows 显示缩放、窗口、无边框和全屏截图。自动化期间游戏必须保持前台，用户不操作鼠标键盘。

### 2.1 非目标

- 不读取或修改游戏内存。
- 不向游戏进程注入代码，不安装驱动，不规避反作弊。
- 不支持自动寻找钓鱼点、移动角色、选择鱼饵或管理背包。
- 不保证普通置顶窗口能覆盖真正的独占全屏；此模式下自动化和 F8 暂停仍需工作，若必须看见控制窗则使用无边框或窗口模式。

## 3. 用户界面

控制窗口默认约 320×240，使用紧凑深色样式，可拖动并保持置顶，默认位于左上角以避开顶部进度条和右下角 F 图标。支持最小化和退出。

窗口包含：

- 游戏窗口状态和“绑定游戏”按钮。
- 1～999 的目标次数输入框；运行时锁定。
- 当前阶段、已完成/目标次数、实际识别帧率、最近错误。
- “开始”“暂停/继续”“重新绑定”“退出”按钮。
- 固定显示“F8 紧急暂停”提示。

点击“绑定游戏”后显示三秒倒计时。倒计时结束时读取前台窗口句柄、客户区屏幕坐标、所属显示器和进程信息。绑定只对当前进程会话有效，重新启动程序后需重新绑定，避免保存失效句柄。

## 4. 总体架构

### 4.1 技术选择

- Python + Tkinter/ttk：控制窗口。
- DXcam：Windows Desktop Duplication 截屏，目标 30 帧/秒。
- NumPy + OpenCV：小区域颜色与形态特征识别。
- ctypes 调用 Win32：DPI、窗口位置、前台状态、全局热键、SendInput、客户区点击和捕获排除。
- PyInstaller：单文件发布。

### 4.2 模块职责

| 模块 | 职责 | 不承担的职责 |
| --- | --- | --- |
| 界面 | 收集目标次数、展示状态、转发按钮命令 | 不直接截图或发按键 |
| 窗口平台层 | 绑定句柄、客户区、显示器、DPI、前台检查 | 不判断钓鱼状态 |
| 截屏层 | 连续获取最新帧、时间戳和实际帧率 | 不做颜色判断 |
| 视觉层 | 从归一化关键区域输出置信度和几何位置 | 不发送输入 |
| 自动化状态机 | 状态切换、超时、次数、暂停恢复 | 不访问界面控件 |
| 输入层 | F/A/D、鼠标点击、按键状态和强制释放 | 不决定按键方向 |
| 存储层 | 非敏感配置、简短日志、异常截图和清理 | 不保存正常连续帧 |

界面运行在主线程。截屏与自动化运行在工作线程，通过线程安全事件和不可变状态快照通信。界面关闭、暂停、异常和进程退出都调用同一套幂等的“释放全部按键”操作。

## 5. 坐标、截屏与适配

程序声明每显示器 DPI 感知。所有识别区域和点击点使用游戏客户区归一化坐标，不使用固定的 2560×1440 像素值。

每次开始和窗口移动后重新计算：

1. 游戏客户区左上角与宽高。
2. 客户区所在显示器和截屏输出编号。
3. 客户区与当前显示器的交集。
4. 顶部进度条、右下角按钮和中央结算区域的实际像素范围。

初始归一化关键区域以客户区左上为 `(0, 0)`、右下为 `(1, 1)`：顶部进度条为 `(0.24, 0.00)～(0.76, 0.15)`，右下角 F 与就绪布局为 `(0.84, 0.68)～(1.00, 1.00)`，中央结算特征为 `(0.25, 0.05)～(0.75, 0.95)`。识别器只在这些区域内寻找满足颜色、形态和空间关系的目标，不能把截图中的红色标注当作目标。窗口位置和客户区尺寸在每次状态动作前检查，并在运行时至少每 0.5 秒复核一次。

截图线程以 30 帧/秒请求最新桌面帧并记录时间戳。视觉层只复制当前状态需要的小区域；控制阶段优先处理顶部区域。工作线程只消费最新帧，不排队补处理旧帧。

置顶窗口在 Windows 10 2004 及以上尝试使用 `WDA_EXCLUDEFROMCAPTURE` 排除出系统截图；失败时窗口仍放在关键区域之外。若游戏跨越多个显示器、客户区被遮挡、最小化或有效尺寸小于 960×540，程序暂停并提示调整窗口。

## 6. 视觉识别

用户提供的截图含红圈、箭头和矩形，仅用于确认位置与流程，不作为完整模板直接匹配。

### 6.1 上钩识别

按第一次 F 后，在右下角 F 图标关键区域建立短期基线。每帧计算蓝色高饱和像素比例、白色图标亮度、环形边缘强度和相对基线变化。至少两项特征超过阈值且连续两帧成立，才判定上钩并按第二次 F。单一亮点或背景运动不得触发。

### 6.2 进度条识别

在顶部归一化区域中分别建立：

- 绿色/青绿色安全区掩膜：按色相、饱和度、亮度与水平连通宽度筛选。
- 黄色标记掩膜：按黄色色相、高亮度、竖向长宽比和与进度条的空间关系筛选。

每帧输出绿色区间 `[green_left, green_right]`、黄色中心 `yellow_x`、各自置信度和帧时间。绿色区域或黄色标记可能快速变化，因此控制决策不得复用上一帧位置。

### 6.3 结算和就绪识别

结算状态需同时满足进度条消失、中央结果卡片的蓝色/暗色覆盖特征出现，并持续至少三帧。点击结算空白区后，只有重新识别到右下角普通 F 图标及钓鱼就绪布局，才认为本轮成功完成并增加计数。

## 7. A/D 闭环控制

控制循环每帧重新计算绿色区域的内缩安全带。安全带左右各内缩绿色宽度的 15%，但至少保留可识别的中心范围。

- `yellow_x` 位于安全带左侧：目标按键为 D。
- `yellow_x` 位于安全带右侧：目标按键为 A。
- `yellow_x` 位于安全带内：释放 A 和 D。

方向切换必须先释放旧键，再按下新键；任何时刻不得同时按 A 和 D。每个按键状态必须受最新帧看门狗约束：一帧未识别到任一目标就立即释放 A/D；连续六帧、约 0.2 秒无法可靠识别则暂停。截屏时间戳超过 0.2 秒未更新也立即释放，超过 0.5 秒则暂停。

## 8. 状态机与超时

| 状态 | 动作 | 成功条件 | 默认超时 |
| --- | --- | --- | --- |
| 未绑定 | 等待绑定 | 获得有效游戏客户区 | 无 |
| 就绪 | 激活游戏并按 F | 输入成功 | 3 秒 |
| 等待上钩 | 监控右下角变化，确认后发送第二次 F | 连续两帧交叉确认且输入成功 | 120 秒 |
| 等待进度条 | 第二次 F 后监控顶部 | 绿色区和黄色标记均出现 | 8 秒 |
| 闭环控制 | 每帧决定 A/D/释放 | 进度条连续两帧消失或结算特征出现 | 120 秒 |
| 等待结算 | 释放 A/D，确认结果卡 | 结算连续三帧成立 | 10 秒 |
| 关闭结算 | 点击客户区左侧安全空白点 | 钓鱼就绪布局重新出现 | 8 秒 |
| 轮间等待 | 更新计数并等待 | 未达到目标则回到就绪 | 1 秒 |
| 暂停 | 释放全部按键 | 用户恢复且画面可重新归类 | 无 |
| 完成 | 释放全部按键并提示 | 已完成数等于目标数 | 无 |

暂停后继续时先重新分类当前画面：若处于可识别的闭环控制或结算状态则恢复对应状态；若已回到钓鱼就绪界面，则从未完成的当前轮重新开始；无法分类时继续保持暂停。

## 9. 输入与安全

所有键盘和鼠标动作通过 Win32 `SendInput` 发送，并检查实际发送数量。若游戏以更高完整性级别运行导致输入失败，程序暂停并提示用户以相同权限启动本程序。

F8 使用 Win32 全局热键注册。若注册失败，程序禁止开始自动化并提示快捷键冲突。窗口暂停、F8、窗口失前台、截图异常、输入失败、状态超时和退出均执行：

1. 释放 A、D、F 和鼠标左键。
2. 停止产生新输入。
3. 保存一次诊断快照和错误代码。
4. 更新界面为暂停状态。

点击结算使用客户区归一化坐标 `(0.15, 0.55)` 的左侧中部空白点，并在点击前再次确认结算状态，避免在普通游戏画面误点。

## 10. 配置、日志与自动清理

程序数据目录为 `%LOCALAPPDATA%\异环自动钓鱼\`：

- `config.json`：窗口位置、最近目标次数和非敏感识别设置。
- `diagnostics/`：异常截图和对应文本日志。

正常连续截图只存在于内存，不写入磁盘。每次程序启动和每次保存异常后执行清理：先删除超过 7 天的诊断文件，再按时间仅保留最近 20 组。清理范围固定在上述诊断目录，绝不处理项目的 `流程截图` 或其他用户目录。

## 11. 测试与验收

所有测试代码、合成帧和回放产物放在 `try/`，删除该目录不影响正式程序。

### 11.1 自动测试

- 归一化坐标在 1280×720、1920×1080、2560×1440 和非 16:9 窗口中的映射。
- 上钩特征的连续帧确认与单特征误触发保护。
- 30 帧/秒合成绿色区域和黄色标记快速移动时的 A/D 方向、换向顺序、进入安全带后的释放。
- 缺帧、旧帧、低置信度、窗口失效、前台丢失和各状态超时。
- 成功轮次计数只在关闭结算并恢复就绪后增加。
- F8 和所有退出路径释放按键。
- 7 天与 20 组诊断文件清理边界，验证不会越出诊断目录。

### 11.2 构建验证

- 开发入口可启动且关闭后无残留工作线程。
- PyInstaller 单文件构建成功。
- 在没有项目虚拟环境的 Windows 会话中启动发布物，完成窗口打开、绑定倒计时和 F8 注册冒烟测试。

### 11.3 实机人工验收

由于仓库没有实时游戏会话，以下项目必须由真实《异环》画面人工确认：窗口、无边框、全屏下的绑定和截屏；真实上钩识别；快速进度条闭环；结算关闭；连续多轮；异常暂停。每项需在 `doc/验收标准.md` 记录步骤、结果、日志或截图证据、问题和最终结论。

## 12. 完成定义

只有满足以下条件才可合并主分支并发布：

1. 自动测试全部通过，构建命令成功。
2. 可执行程序完成启动冒烟测试。
3. 代码不存在第二套重复状态机或临时固定坐标补丁。
4. `AGENTS.md`、验收标准和进展记录与最终实现一致。
5. 实机项目明确区分已通过与待人工确认；不得把未执行的实机测试写为通过。
