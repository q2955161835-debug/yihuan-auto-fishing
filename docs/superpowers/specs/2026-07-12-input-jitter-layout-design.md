# 输入节奏与控制窗口布局设计

## 目标

在不改变安全输入边界和 A/D 闭环响应性的前提下，为 F 轻点加入短、受限的操作延迟，并消除控制窗口右侧文本被裁切的问题。

## 范围

- 仅 F 轻点加入 80～180 毫秒的均匀随机延迟，延迟发生在鼠标按下前。
- A/D 的按下、换向和释放不加入随机延迟，保持每帧即时响应。
- 暂停、F8、输入阻断或关闭发生在延迟期间时，取消尚未发送的 F 输入。
- 控制窗口最小宽度由 320 像素增加到 400 像素，并为右侧状态与帧率标签留出水平间距。

## 非目标

- 不为规避游戏保护或检测而模拟人类行为。
- 不改变进度条识别、A/D 方向决策、超时阈值或屏幕键盘按键坐标。
- 不加入可由用户调节的随机延迟参数，避免增加误配置面。

## 实现与测试

输入服务提供可注入的延迟/随机源，使单元测试可固定边界值。测试覆盖 F 延迟范围、A/D 零额外延迟、暂停时取消待发送 F，以及窗口几何与右侧控件不会被裁切。全量测试、单文件构建和发布物清单验证仍是交付门槛。
