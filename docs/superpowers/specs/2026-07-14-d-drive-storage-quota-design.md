# D 盘数据迁移与 100 MiB 配额设计

## 目标

将《异环自动钓鱼》的全部生产数据从
`C:\Users\29551\AppData\Local\异环自动钓鱼\` 迁移到
`D:\29551\异环自动钓鱼数据\`，并保证新数据根目录在稳定写入后不超过
100 MiB（104,857,600 字节）。容量统计包含运行日志、低分辨率逐帧截图、
异常诊断、配置和目录中的其他普通文件。

迁移和配额改动不得影响已验证的钓鱼状态机、画面识别、屏幕键盘输入、
F8 暂停和结算流程。

## 固定约束

- 新数据根目录固定为 `D:\29551\异环自动钓鱼数据\`。
- 整个数据根目录的硬上限为 100 MiB，不使用十进制 100 MB。
- `config.json` 始终保留，不作为自动删除候选。
- 正常写入允许在单个文件落盘的瞬间短暂超过上限；该次写入返回前必须完成
  清理并恢复到上限以内。
- 不引入环境变量、注册表或用户可编辑路径设置。
- 不降低运行截图最长边 480 像素和 JPEG 质量 50 的现有取证质量。
- 数据迁移属于一次性发布操作；生产程序直接使用新路径，不在每次启动时重复
  扫描或搬运旧 C 盘目录。

## 方案选择

采用增量严格配额，而不是只在启动/退出时清理或定时全目录扫描。

- 启动装配时完整扫描一次新数据根目录，建立当前字节数并立即清理超额数据。
- 正常写入者把写入前后文件大小差额报告给共享配额管理器。
- 未超限时只更新内存账本，不遍历目录。
- 超限时重新核算目录并按删除顺序回收空间；清理完成后再次核算，确认总量不高于
  104,857,600 字节。
- 配额管理器用互斥锁串行化运行日志线程、诊断保存和配置保存的容量更新与清理。

## 模块边界

新增 `src/auto_fishing/storage/quota.py`，只负责数据根目录容量核算和删除策略。

核心接口：

```python
class StorageQuotaError(RuntimeError): ...

class StorageQuotaManager:
    def __init__(
        self,
        root: Path,
        *,
        max_bytes: int = 100 * 1024 * 1024,
    ) -> None: ...

    def initialize(self) -> None: ...

    def register_write(
        self,
        path: Path,
        previous_size: int,
        *,
        active_run: Path | None = None,
        active_events: Path | None = None,
    ) -> None: ...
```

`initialize()` 在任何生产数据写入前执行。`register_write()` 验证写入路径仍位于数据
根目录，更新增量账本，并仅在超限时执行清理。无法恢复到上限以内时抛出
`StorageQuotaError`，现有应用错误边界必须安全暂停自动化，不得静默无日志继续运行。

现有存储模块的职责保持不变：

- `SettingsStore` 仍只负责原子读写 `config.json`，写入完成后报告大小变化。
- `DiagnosticsStore` 仍按错误组写入 PNG、JSON 和联系图，并保留原 7 天/20 组
  清理作为第一层约束；组写入完成后报告每个文件的大小变化。
- `RuntimeLogStore` 仍异步写入 `events.jsonl` 和 `frames/*.jpg`；每次写入完成后从
  日志写线程报告大小变化，并把当前运行目录与事件文件交给配额管理器。
- 应用装配创建一个共享 `StorageQuotaManager`，依次完成初始化，再注入三个存储服务。

## 删除顺序

超限时按以下顺序回收，直到整个数据根目录不超过 100 MiB：

1. 删除 `runs/` 中最旧的已结束运行目录，当前活动运行目录永不在此步删除。
2. 按诊断组最新修改时间删除 `diagnostics/` 中最旧组；同一 stem 的 PNG、JSON 和
   `_progress.jpg` 作为一个原子组删除。
3. 删除当前活动运行 `frames/` 中编号最小、时间最早的 JPEG，保留最近帧。
4. 如果仅靠上述数据仍无法达标，原子重写当前 `events.jsonl`，删除最早的完整行，
   保留能放入剩余预算的最新完整 JSON 行；至少保留最后一条事件。

`config.json`、活动运行目录结构和当前 `events.jsonl` 文件本身不得被删除。未知普通
文件计入容量但不自动删除；如果未知文件加配置本身已超过上限，抛出
`StorageQuotaError` 并停止继续写入。

旧的 30 个运行目录上限继续保留；运行目录数量或总容量任一超限时都进行清理。

## 迁移流程与回退

迁移由本次发布过程执行，不由游戏自动化线程执行：

1. 确认应用和日志写入线程均已关闭。
2. 将当前 C 盘数据完整复制到带时间戳的
   `D:\0文件夹\备份\异环自动钓鱼-data-migration-YYYYMMDD-HHmm\`；同类备份最多
   保留两份。
3. 对备份执行相对路径、文件大小和 SHA256 全量清单校验。
4. 将 C 盘数据复制到 `D:\29551\异环自动钓鱼数据\`，再执行同一全量清单校验。
5. 使用新生产配额管理器对 D 盘目标执行清理，确认整个目录不超过 100 MiB、
   `config.json` 存在且可读取。
6. 构建新 EXE，完成测试、清单和烟雾验证后，才删除旧 C 盘数据目录。
7. 删除 C 盘目录前再次验证解析后的绝对路径严格等于旧数据根目录；禁止对其父目录
   或通配路径执行删除。

回退时停止新程序，删除或改名 D 盘目标，从带时间戳备份恢复旧 C 盘目录，并恢复
迁移前 EXE。代码使用 Git 提交回退，不使用数据备份替代版本管理。

## 测试与验收

自动测试必须先失败再实现，并至少覆盖：

- 应用装配使用固定 D 盘路径，不再读取 `LOCALAPPDATA`。
- 容量统计包含运行日志、帧、诊断、配置和未知普通文件。
- 未超限时不删除任何文件。
- 超限时先删除最旧已结束运行，再删除最旧诊断组。
- 仅活动运行过大时删除最早帧并保留最新帧。
- 事件文件过大时只保留最新完整 JSON 行，配置始终保留。
- 未知不可删除文件导致无法达标时抛出 `StorageQuotaError`。
- 所有删除路径都被限制在数据根目录，路径逃逸被拒绝。
- 现有 30 运行目录、7 天/20 诊断组、异步写入失败暂停行为继续通过。

发布验收：

- 全量 pytest 通过，`compileall` 和 `git diff --check` 通过。
- 迁移前备份、C→备份和 C→D 两份全量 SHA256 清单一致。
- 配额清理后 D 盘整个数据目录不超过 104,857,600 字节。
- 根目录 `异环自动钓鱼.exe` 构建、管理员清单校验和烟雾通过。
- 本地功能分支合并 `main` 后再次执行全量测试。
- 删除功能工作树与分支，确认 `main` 工作区干净，再推送 `origin/main`。

- 不自动上传、压缩或加密日志。
