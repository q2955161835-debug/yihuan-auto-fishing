# 异环自动钓鱼项目规则

## 项目状态

- 当前阶段：V1 核心自动钓鱼闭环已通过用户真实游戏验收并留存为 GitHub `v1.0.0`。V2 的 GitHub 最新发布为 `v2.0.4`；用户已完成真实游戏验证并确认其为当前最稳定版本。该版本包含绿色连续段修复、Windows 系统弹窗/控制窗口抢占前台时的输入守卫、明确暂停分类和干净退出处理，完成 428 项自动测试；180 张最新诊断进度槽回放和 412 帧历史控制回放通过，单文件构建及管理员/DPI 清单校验通过。V2 正常运行只保存设置，不持续写运行日志；异常或用户主动报告时生成最多 5 份诊断 ZIP。Windows 11 已验证，Windows 10 真机仍待人工确认。
- 项目目标：提供 Windows 单文件可执行程序，通过画面识别和系统输入完成《异环》钓鱼循环。
- 使用前提：自动化期间游戏保持前台，用户不操作鼠标键盘；F8 为全局紧急暂停键。
- 范围边界：不读取游戏内存、不注入游戏进程、不规避反作弊，只使用屏幕截图和 Windows 标准输入接口。
- GitHub：公开仓库 `https://github.com/q2955161835-debug/yihuan-auto-fishing`；默认分支为 `main`，当前本地 `main` 跟踪 `origin/main`。
- 对外说明与授权：根目录 `README.md` 是安装、使用、诊断与源码构建入口；`LICENSE` 使用 PolyForm Noncommercial License 1.0.0，仅授权非商业用途，且本项目不属于 OSI 认可的开源软件。

## 技术栈与关键依赖

- Python 3.13.x：桌面程序与自动化逻辑；本次离线构建环境为 Python 3.13.5。
- Tkinter/ttk：小型置顶控制窗口。
- DXcam：Windows Desktop Duplication 高频截屏，目标 30 帧/秒。
- NumPy 与 OpenCV：关键区域颜色分割、连通区域和画面特征识别。
- Win32 ctypes：窗口绑定、客户区坐标、前台检查、DPI 适配、全局 F8、SendInput 和捕获排除。
- PyInstaller：生成单文件 Windows 可执行程序。
- pytest：单元、状态机和合成帧回放测试。

运行依赖固定为 DXcam 0.3.0、NumPy 2.4.1、opencv-python-headless 4.13.0.92；开发依赖固定为 pytest 9.1.0 和 PyInstaller 6.19.0。变更截屏、输入或打包依赖时，必须复测窗口、无边框和全屏模式。

## 架构与数据流

已实现模块边界如下：

- `src/auto_fishing/ui/`：窗口、状态展示和用户操作，不包含识别与按键策略。
- `src/auto_fishing/platform/`：Windows 窗口、DPI、热键、鼠标输入接口和系统屏幕键盘生命周期/按键适配。
- `src/auto_fishing/capture/`：显示器选择、30 帧/秒截图和客户区裁剪。
- `src/auto_fishing/vision/`：上钩、进度条、黄色标记、绿色区域、结算和就绪画面识别。
- `src/auto_fishing/automation/`：状态机、超时、循环计数、暂停与放弃当前未完成轮后重开。
- `src/auto_fishing/storage/`：共用配置和事件字段；V1 使用异常诊断、完整运行记录与 `StorageQuotaManager` 增量容量账本，V2 使用统一 30 秒的事件、进度识别轨迹、低分辨率上下文帧和无损顶部进度槽内存环，并按需生成诊断 ZIP，正常运行不持续落盘。
- `packaging/`：`requireAdministrator`、`PerMonitorV2` 清单和 PyInstaller 单文件规格；规格文件必须同时设置 `uac_admin=True`，否则 PyInstaller 会把自定义清单重写为 `asInvoker`。发布物启动时请求 UAC（用户账户控制）提升，不得启用控制台子系统。
- `scripts/build.ps1` 与 `scripts/build_v2.ps1`：默认从项目 `.venv` 运行完整测试后分别构建 V1/V2 单文件发布物并输出 SHA256；干净环境验收可用 `-PythonPath` 参数或 `AUTO_FISHING_PYTHON` 环境变量指定解释器。
- `scripts/verify_release.py`：读取最终 EXE 的 `RT_MANIFEST` 内嵌资源，构建后强制验证 `requireAdministrator`、`uiAccess=false`、`PerMonitorV2` 和 `true/pm` 旧系统回退声明，不得只检查源 XML。
- `try/`：测试、合成帧、回放输入和临时产物；删除后不得影响正式程序。
- `try/smoke_exe.ps1`：以启动器 PID 为根递归跟踪所属进程树，检查窗口响应并只关闭本次烟雾拥有的进程；强制结束后按条件等待进程消失，不得按可执行文件路径批量判定或终止进程。脚本在启动前强制检查当前 PowerShell 已提升，避免中等完整性测试壳无法关闭管理员发布物并遗留进程；V2 用 `-TargetPath dist\异环自动钓鱼V2.exe` 指定候选。
- `流程截图/`：用户提供的带标注流程参考图，不作为可直接匹配的干净模板。
- `doc/验收标准.md`：真实使用流程、验证证据和结论。
- `doc/进展记录/`：按日期记录阶段性修改和异常。
- `docs/superpowers/specs/`：已确认的设计规格。
- `docs/superpowers/plans/`：按测试先行拆分的实施计划。

主数据流：点击“绑定并开始” → 在唯一一次 3 秒倒计时内手动切到游戏 → 绑定游戏窗口 → 启动或复用 Windows 屏幕键盘并固定到游戏显示器左下角 → 立即启动自动化 → 获取客户区与显示器 → 30 帧/秒截图 → 识别器返回结构化结果 → 状态机决定输入 → 鼠标点击/按住屏幕键盘 F/A/D → 顶部进度条干净消失后随机计时并点击游戏安全空白处 → 成功计数 → 轮间等待 1 秒 → 自动抛出下一轮。每次新的 F/A/D 按下或结算点击都必须在物理输入前重新确认已绑定游戏仍为前台；前台丢失后禁止新输入，但抬起按键/鼠标的释放动作始终允许。控制窗口取得前台按 `E_USER_PAUSE` 安全暂停且不自动生成诊断，Windows 系统弹窗或其他窗口抢占按 `E_WINDOW` 安全暂停并生成诊断；均不得自动恢复，用户关闭弹窗后点击“继续”，放弃当前未完成轮并从第一 F 重开。V1 每帧异步写完整运行记录；V2 仅在内存保留最近 30 秒事件、每个已处理进度帧的识别轨迹、最高 10 FPS 的 480 像素 JPEG 和最高 10 FPS 的原生宽度无损顶部进度槽 PNG，异常或“报告错误”时冻结快照并异步生成 ZIP。已绑定后的“开始”和暂停后的“继续”默认显式激活游戏窗口；“继续”保留目标数与已完成数，但放弃当前未完成鱼并从第一 F 开启新一轮，不再根据单帧画面恢复到上钩、进度或结算中间阶段。关闭“自动切回游戏”后恢复 3 秒手动切回倒计时。任何自动化异常先抬起鼠标并释放 A/D，再暂停。

游戏不接受程序直接发送的键盘 `SendInput`，但真实验证接受实体鼠标点击 Windows 屏幕键盘生成的按键。生产输入后端必须保持屏幕键盘可见，通过客户区归一化坐标定位 F/A/D：F 轻点，A/D 以鼠标左键按住，换向先抬起再移动并按下。每次绑定必须按固定外框比例 `1365:415` 重建尺寸，目标宽度同时受显示器宽度、1365 像素、游戏客户区宽度三分之二和右下准备区可见边界约束，1280×720 等低分辨率下不得遮挡上钩/准备识别区；不得继承屏幕键盘当前尺寸。绑定后及每次点击前必须重新读取客户区，宽度低于 900、高度低于 250 或宽高比不在 `3.2～3.9` 时拒绝点击并以 `E_OSK` 暂停，防止 Windows 响应式布局导致归一化坐标落到错误按键。所有原生 `user32` 函数必须声明准确的 `argtypes/restype`，尤其 `SetWindowPos` 的两个 `HWND` 参数必须为指针宽度，避免错误 1400。`osk.exe` 必须通过 `ShellExecuteW` 的普通 `open` 动作启动；直接子进程创建会触发 Windows 错误 740。发布物经用户授权请求管理员权限，以跨越屏幕键盘 `UIAccess`（用户界面访问）完整性边界；同时保留错误 5 降级：读取并验证现有位置，安全则继续，否则要求用户手动拖到左下角。已有屏幕键盘退出时保留；本程序启动的实例在释放输入后尽力关闭，若 Windows 拒绝关闭则安全保留，不得强制终止系统辅助功能进程；关闭时若句柄已失效并返回错误 1400，按“已由 Windows 关闭”幂等成功处理。屏幕键盘消失、越界或遮挡顶部进度/右下角准备区域时以 `E_OSK` 安全暂停；不得退回直接扫描码作为静默降级。

真实《异环》上钩提示位于游戏客户区右下 `BITE_ROI=(0.89,0.79)-(0.99,0.99)` 的蓝白圆环，左上文字提示不是可用的上钩信号。第一次 F 前以该区域建立初始基线；前 45 帧（约 1.5 秒）忽略抛竿动画，同时只允许把蓝色基线降低到冷却期实际背景的低位，不得继续使用可能含右下准备鱼钩按钮的高蓝色单帧。冷却后要求蓝色比例相对动态低位基线增加大于 `0.03`，且白色比例增加大于 `0.03` 或 Canny 边缘比例增加大于 `0.02`，连续两帧才确认；白色和边缘基线保持抛竿前值以避免放宽形状门槛。顶部进度识别固定使用 `TOP_ROI` 局部纵向 40%～52% 的五条扫描线，并在排除两侧固定 HUD 图标后的整个顶部槽带可玩区分别检测黄色标记和绿色目标区；两者不得再以相邻、重叠或固定距离作为成立条件。黄标切断绿区时仍重建完整逻辑绿区，黄标在绿区外时则保留独立绿段；至少三条扫描线边界一致才输出观测。最小逻辑绿区宽度为 `max(4 像素, TOP_ROI 宽度 * 0.012)`，不得再乘以黄标宽度；有历史时以最近可靠的绿区与黄标位置分别对同色候选排序，但达到最小宽度的绿区必须优先于附近的微小噪声，历史观测本身不得直接驱动输入。第二次 F 成功并进入等待进度条后，首张后续帧建立 0.60 秒启用门；门内忽略全部进度候选、持续清空绿区/黄标历史并释放 A/D，门后只有连续 3 张时间戳递增且帧间隔不超过 0.20 秒的完整观测才能进入控制。确认缺失、时间戳不递增或间隔超限时从干净历史重新确认。该入口门不得改变短绿区最小宽度，也不得约束黄标和绿区相对位置。新阶段首个可靠黄标可位于可玩区任意位置，不得受上一轮或画面中心约束。进入控制后，每张达到至少三条扫描线共识的可靠观测必须在当前帧立即参与控制；不得因绿区中心或黄标相对历史位置跳变而等待下一帧确认或释放 A/D。控制目标是让黄色标记回到绿色区域中心：每帧使用 `(黄标位置 - 绿区中心) / 绿区宽度` 作为相对误差，只保留连续且有效的最近 15 帧；每个样本的权重为识别置信度乘以 `0.2^帧龄`，同等清晰度时最新一帧可压过此前 14 帧。加权误差小于 `-0.10` 时按 D，大于 `0.10` 时按 A，其余释放；不得引入速度预测、旧绿区绝对位置或第二套控制状态机。任何缺失帧、非递增时间戳或超过 0.20 秒的帧间隔都必须立即清空控制窗口并释放 A/D；每个有效决策将帧时间戳、绿区/黄标坐标、瞬时与加权误差、方向、样本数、置信度、请求按键和输入切换前状态写入 `progress.control` 运行事件。结构性歧义和完全空白均最多容忍 60 帧，任一类别恢复可靠观测后计数归零，类别切换从 1 重新计数。累计至少 15 个可靠控制帧后，若连续三帧同时满足无进度观测、零扫描线、零候选且拒绝原因为 `yellow_missing`，仍优先通过现有 `BAR_GONE` 进入等待结算；中央提示横幅只作为运行诊断字段，不得触发额外 F 或提前进入结算。

等待结算不得依赖鱼种、品质、卡片颜色或其他结算页视觉特征。累计至少 15 个可靠控制帧且顶部进度条连续三帧干净消失后，从消失确认时刻随机等待 3.10～3.60 秒，再选择未被屏幕键盘遮挡的游戏客户区安全空白点点击一次。该窗口是在 2.10～2.60 秒实机点击失败后按用户要求整体后移 1 秒；历史运行 `run-20260713T070804371619Z` 仅作为初始量测证据，不再视为当前可用时机。输入服务点击正常返回后立即记录 `signal=click_succeeded`，把当前轮计为成功并进入完成或轮间等待，不再识别结算页、卡片消失、右下初始鱼钩或其他结束画面，也不重试；点击抛错时不得计数，必须走既有 `E_INPUT` 安全暂停。所有候选点被屏幕键盘遮挡时以 `E_OSK` 暂停。随机计划、尝试坐标、计时触发和点击成功必须写入完整运行日志；等待结算期间暂停后点击“继续”会放弃当前未完成轮并重新抛竿，不再保留或重抽当前结算计时。

Tk 控制窗口的内部句柄必须先通过 `GetAncestor(..., GA_ROOT)` 解析为顶层 HWND；自窗口识别与 `WDA_EXCLUDEFROMCAPTURE` 必须使用同一个顶层句柄。捕获排除失败属于非致命警告，必须显示在控制窗口中。暂停且没有恢复令牌时，worker 必须在截帧、窗口前台/刷新和视觉识别前等待；调用 `latest()` 前保留 operation epoch，取得帧后保留 frame epoch，截图异常、陈旧帧释放失败和后续帧处理错误都只能在对应 epoch 仍有效时触发暂停。迟到错误不得清除新恢复令牌或退出可继续运行的 worker。只有用户直接点击已绑定的“开始”或“继续”且“自动切回游戏”开启时允许调用窗口激活；初次“绑定并开始”和每轮自动抛竿不得激活窗口。激活失败或激活后仍非前台时不得发送输入。F8 必须使待绑定并开始、待开始和待继续倒计时失效，且从热键线程向 Tk 交付取消提示时必须走主线程队列。窗口失效后使用“取消当前轮 → 重新绑定”路径，不得把引擎永久关闭。关闭控制窗口时必须先停止控制器和输入，再保存设置并销毁 Tk 窗口，避免设置写入期间 worker 仍在运行。

顶部进度槽的固定可玩区在 `TOP_ROI` 内只排除左右各 14%的 HUD 区域；真实槽左端不得再被 16%旧边界裁掉。黄色掩码覆盖 HSV `H=18～50、S=40～255、V=120～255`，低饱和度候选还必须覆盖扫描带至少 35%的纵向像素并保持窄竖条，避免仅靠放宽颜色吸入背景。绿色使用两层掩码：宽松边界掩码覆盖 `H=70～105、S=80～255、V=100～255`，只用于恢复被黄标切断的逻辑外边界；实色掩码覆盖 `H=70～105、S=155～255、V=100～255`，用于证明横向连续绿色。宽松候选获得的实色累计支撑必须不小于现有最小逻辑绿区宽度，独立候选只能来自实色连续段；不得通过提高最小宽度排除背景，以免漏掉高品质短绿区。真实诊断包 `yihuan-v2-20260714T145041695950Z-MANUAL_REPORT.zip` 的 1280×720 原图必须识别出绿区约 `0.572～0.715`、黄标约 `0.147` 并输出 D；`yihuan-v2-20260715T022333974433Z-MANUAL_REPORT.zip` 第 1117 帧必须识别出左移绿区约 `0.198～0.341`、黄标约 `0.851`，第 1121 帧后的蓝灰背景不得输出假绿区。

## 核心功能

- 对外只分发单文件 EXE；用户直接双击后由 EXE 内嵌 `requireAdministrator` 清单触发 UAC，不要求用户打开管理员命令行或运行任何脚本。提升 PowerShell 只用于开发烟雾测试。
- “绑定并开始”在一次三秒倒计时后完成绑定并直接启动，用户只需切换游戏一次。
- 已绑定的开始和暂停后的继续默认自动激活游戏；继续保留完成数和目标数、放弃当前未完成轮并重新抛竿；设置可关闭并恢复三秒手动切回倒计时。
- 设置 1～999 次钓鱼循环。
- 通过左下角 Windows 屏幕键盘自动完成抛竿、等待上钩、收杠、A/D 闭环控制；关闭结算和下一轮继续使用游戏画面坐标点击。
- F 轻点在按下前随机等待 80～180 毫秒，并在 `release_all()`（释放所有输入）期间取消尚未发送的 F；A/D 长按、换向和释放不允许增加该延迟。该节奏仅为操作稳定性设计，不得声称或用于规避游戏保护。
- 支持不同分辨率、Windows 缩放、窗口、无边框和全屏截图。
- 控制窗口始终置顶；真正独占全屏下不保证可见，但自动化与 F8 暂停仍应工作。
- V1 控制窗口初始与最小尺寸为 400×240 像素；V2 为 400×300 像素并显示“报告错误”“打开文件位置”和诊断包路径。右侧“阶段/帧率”列必须保留左侧间距，错误文本按 305 像素换行，避免裁切。
- 识别失败、窗口异常、截图停滞、输入失败或 F8 时安全暂停。
